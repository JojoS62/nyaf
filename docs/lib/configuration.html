linkConfiguration

A @nyaf application consists of:

  1. An entry file for registering components, typically called main.ts
  2. At least one root component
  3. The index.html file the browser loads first
  4. The configuration for TypeScript, tsconfig.json
  5. The Packer / Builder setup;
    • The best choice is probably WebPack, in that case a webpack.config.js file is recommended

To get a new project quickly you can use Yeoman and the various generators explained here

linkTypeScript Configuration

The TypeScript configuration is typical, but two things are crucial to know:

  1. You need to compile with the target "es2015" (minimum). ES 5 is explicitly not supported anymore.
  2. The template language is a variety of JSX, so the setting *"jsx"* and *"reactNamespace"* are required.

@nyaf does not use React, has no relation to React and has almost nothing in common. The setting just tricks the compiler to transpile the templates.

1{

2  "compilerOptions": {

3    /*!*/ "target": "es2015",     // The target must be at least ES 2015

4    "module": "commonjs",

5    "moduleResolution": "node",

6    "resolveJsonModule": true,

7    "sourceMap": true,

8    "lib": ["es2018", "es5", "dom"],

9    /*!*/ "jsx": "react",           // tell the compiler to accept JSX

10    /*!*/ "reactNamespace": "JSX",  // the name of the support class in @nyaf (this IS mandatory)

11    "declaration": false,

12    "experimentalDecorators": true,

13    "noImplicitAny": false,

14    "suppressImplicitAnyIndexErrors": true,

15    "removeComments": false,

16    "outDir": "out-tsc",

17    "baseUrl": "./src",

18    "typeRoots": [

19      "node_modules/@types",

20      "src/types"

21    ]

22  },

23  "files": ["./src/main.ts"]

24}

linkWebPack Configuration

WebPack is the recommended packer tool, but you can use any other if you like. There is no dependency.

A typical configuration will look like this:

1const dev = process.env.NODE_ENV === 'dev';

2const path = require('path');

3const HtmlWebpackPlugin = require('html-webpack-plugin');

4const CopyWebpackPlugin = require('copy-webpack-plugin');

5

6const tsconfig = require('./tsconfig.json');

7

8// Main entry point

9const indexConfig = {

10  template: './src/index.html',

11  inject: 'body',

12  baseHref: './'

13};

14

15const webpackConfig = {

16  mode: 'development',

17  // How source maps are generated : style of source mapping

18  devtool: dev ? 'eval-cheap-module-source-map' : false,

19  // Development server configuration

20  devServer: {

21    historyApiFallback: true,

22    contentBase: path.join(__dirname, 'dist'),

23    compress: true,

24    port: 9000

25  },

26  // Where webpack looks to start building the bundle

27  entry: {

28    'app': './src/main.ts' // Demo app entry point

29  },

30  // How the different types of modules within a project will be treated

31  module: {

32    rules: [

33      { test: /\.ts|\.tsx$/, loader: 'ts-loader' },

34      // All files with a '.scss' extension will be handled by sass-loader

35      {

36        test: /\.(scss)$/,

37        use: [

38          'style-loader',

39          'css-loader',

40          'sass-loader'

41        ]

42      },

43      {

44        test: /\.css$/,

45        use: ['style-loader', 'css-loader']

46      },

47      {

48        test: /\.(png|jpeg)$/,

49        loader: 'url-loader?limit=8192'},

50      {

51        test: /\.(woff|woff2)$/,

52        loader: 'file-loader'

53      }

54    ]

55  },

56  // Configure how modules are resolved

57  resolve: {

58    extensions: ['.ts', '.tsx', '.js', '.scss']

59  },

60  // How and where webpack should output bundles, assets and anything else

61  output: {

62    path: path.resolve('./wwwroot'),

63    filename: '[name].js'

64  },

65  // What bundle information gets displayed

66  stats: {

67    warnings: false

68  },

69  // Target a specific environment (cf. doc)

70  target: 'web',

71  // Configure whether to polyfill or mock certain Node.js globals and modules

72  node: {

73    __dirname: false

74  },

75  // Customize the webpack build process with additionals plugins

76  plugins: [

77    new HtmlWebpackPlugin(indexConfig),

78        new CopyWebpackPlugin([

79            {

80                from: './src/assets/images',

81                flatten: true,

82                to: './assets/images/'

83      }

84    ])

85  ]

86};

87

88// Export the config

89module.exports = webpackConfig;
