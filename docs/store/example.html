linkType Handling in Typescript

The store has these basic parts as described before:

  • Actions
  • Reducer
  • Store and Store Types

The Actions are basically string constants. The reducers get payload that's anything. The return value is the Store Type.

The store has two basic function:

  • dispatch
  • subscribe

You dispatch an Action along with a payload. So, the types are string and any.

When you receive a store event from a subscribe this subscription watches for changes of a part of the Store Type. The event handler receives the whole store, then.

Also, some reducers doing the hard work:

1import { ALL, ADD, REMOVE, EDIT, SAVE } from '../actions/archive.actions';

2import { archiveStoreType } from '../stores/archive.store';

3import { DatabaseService } from 'app/services/database.service';

4import { DataGridModel } from 'app/components/shared/grid/models/datagrid.model';

5import { ArchivModel } from '../model/archiv.model';

6

7import * as $sql from 'app/resources/sql.json';

8

9/**

10 * The reducer functions are the executing logic. They "do" what the action is asking for.

11 */

12export default {

13    [ALL]: async (state: archiveStoreType, payload: string) => {

14        const data: any = await DatabaseService.instance.instance.exec($sql.ArchivAnzeigen);

15        const modelData = new DataGridModel<ArchivModel>(data, ArchivModel);

16        state.gridResult = modelData;

17        return state;

18    },

19    [EDIT]: async (state: archiveStoreType, payload: number) => {

20        const [current]: any = await DatabaseService.instance.instance.exec($sql.ArchivAnzeigenAktuelles, payload);

21        state.current = current;

22        return state;

23    },

24    [ADD]: (state: archiveStoreType, payload: ArchivModel) => {

25        state.current = null;

26        return state;

27    },

28    [SAVE]: async (state: archiveStoreType, payload: ArchivModel) => {

29        const data: any = await DatabaseService.instance.instance.exec(

30            payload.id ? $sql.ArchivUpdate : $sql.ArchivInsert,

31            payload.Schrank,

32            payload.Vorname,

33            payload.Nachname,

34            payload.Geburtsdatum,

35            payload.AnzahlAkten,

36            payload.ArchivJahr,

37            payload.id

38        );

39        const modelData = new DataGridModel<ArchivModel>(data, ArchivModel);

40        state.gridResult = modelData;

41        return state;

42    },

43    [REMOVE]: async (state: archiveStoreType, payload: number) => {

44        const data: any = await DatabaseService.instance.instance.exec($sql.ArchivEntfernenFinal, payload);

45        const modelData = new DataGridModel<ArchivModel>(data, ArchivModel);

46        state.gridResult = modelData;

47        return state;

48    }

49};

DatabaseService.instance.instance is a service class with singleton pattern. It executes SQL. $sql provides the statements from a resource file.

THe store summarizes all this for easy processing:

1import { ArchivModel } from "../model/archiv.model";

2import { DataGridModel } from "app/components/shared/grid/models/datagrid.model";

3

4export interface ArchiveStore {

5    current: ArchivModel;

6    gridResult: DataGridModel<ArchivModel>;

7}

8

9/**

10 * A store contains a data structure that helds up to the entire app's state.

11 * It can have any complexity, from a single value up to deep nested objects.

12 */

13export interface ActionStore {

14  search: string;

15}

16

17/**

18 * We export a single store type that contains all single stores as one default.

19 */

20export type archiveStoreType = ActionStore & ArchiveStore;

21

22import archiveReducer from '../reducer/archive.reducer';

23import archiveActions, { SEARCH, ADD, REMOVE, ALL, ARCHIVED, EDIT } from '../actions/archive.actions';

24import { Store } from "@nyaf/store";

25

26const store = new Store<archiveStoreType>({

27  actions: archiveActions,

28  reducer: { ...archiveReducer },

29  state: { search: '', current: null, gridResult: null }

30});

31

32export default store;

Now, the component can dispatch actions with payloads and receive store changes.

1@CustomElement("tab-archive-search")

2@ProvideStore<archiveStoreType>(store)

3export class ArchiveSearchComponent extends StoreComponent<archiveStoreType, {}> {

4

5    constructor() {

6        super();

7        this.store.subscribe('gridResult', (data: archiveStoreType) => {

8      // Do something with the data

9        });

10    }

11

12     private async showAll(e?: Event) {

13        this.store.dispatch(ALL, null);

14    }

15

16  // render omitted for brevity

17}

The reducer receives the ALL action. It pulls all the data and sets the gridResult object. The subscriber listens for this and can handle the data (re-render, for example).

The essential part is here that the return value of the subscriber is always the Store Type (here archiveStoreType). So you don't need to think about the current type and TypeScript resolves the types within properly. However, the subscriber is for just one property of the store and only changes of this property will trigger the handler. To get the data, access it like this:

1archiveStoreType.gridResult

The underlying object is Proxy, not your actual type.
