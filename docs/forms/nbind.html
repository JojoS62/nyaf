linkTemplate Language Enhancements

@nyaf has a simple template language. For forms it's just one more command for any input element, n-bind:

1model: ModelBinder<UserViewModel>; // instance created by decorator

2

3async render() {

4  const model: UserViewModel = new UserViewModel(); // or where ever the model comes from

5  return await (

6    <>

7      <form>

8        <input n-bind="value: Name" />

9      </form>

10    </>);

11}

Now the field knows everything about how to render and how to validate. The first item ("value") is the HTML element's property you bind to. The second is the model's property name ("Name").

Once you retrieve the DOM object, you get access to validation:

1const username = this.querySelector('[name="userName"]');

2if (username.valid){

3  // call server

4}

5if (username.touched){

6  // show help

7}

For a good UI you need a label usually:

1<label n-model="UserViewModel" name="@userName" />

The model is provided by a decorator

1@ViewModel(ModelType)

2export class component extends BaseComponent<any> implements IModel<ModelType> {

3

4  async render() {

5    return await (

6      <form>

7        <label n-bind="innerText: userName" for="un"/>

8        <input n-bind="value; userName" id="un" />

9        <br />

10        <label n-bind="innerText: city" for="city"/>

11        <input n-bind="value: @city" id ="city" />

12     </form>

13    )

14  }

15

16}

Forms bind data. It's bi-directional or uni-directional depending on the chosen handler.

linkValidation

The error message is just regular output (class example from Bootstrap,not needed by NYAF forms):

1<form>

2  <label n-bind="innerText: userName" for="un"/>

3  <input n-bind="value: userName" id="un">

4  <div class="text text-danger" n-if="!@userName.valid && @userName.touched">Oops, something got wrong</div>

5</form>

Again, note the @ signs preceding the property names.

Validators can provide the error text, too:

1<form>

2  <label n-bind="innerText: userName" for="un"/>

3  <input n-bind="value: userName" id="un">

4  <div class="text text-danger"

5       n-if={this.model.getScope().userName.valid && this.model.getScope().userName.touched"

6       innerHTML={this.model.getScope().userName.errors} ></div>s

7</form>

Distinguish between different validators like this:

1<form>

2  <label n-bind="innerText: userName" for="un"/>

3  <input n-bind="value: userName" id="un">

4  <div class="text text-danger"

5       n-if={this.model.getScope().userName.valid && this.model.getScope().userName.touched"

6       innerHTML={this.model.getScope().userName.errors.required.error} ></div>s

7</form>

Objects are always set (not undefined), so you don't must test first. The property names are same as the decorators, but in lower case:

  • @StringLength: stringlength
  • @MaxLength: maximum
  • @MinLength: minimum
  • @Pattern: pattern
  • @Range: range
  • @Required: required
  • @EMail: email
  • @Compare: compare
